# Exp 008: 5-Fold + 384 Resolution

## 설정

- exp006과 동일한 구조 (ConvNeXt-Small, 5-Fold, train+dev 1,100샘플)
- 핵심 변경: IMG_SIZE 224 → 384 (해상도 약 2.9배 증가)
- batch_size: 8→4, grad_accum: 4→8 (실효 배치 32 유지, GPU 메모리 대응)
- 추론: 5-Fold x 5-TTA = 25개 예측 평균

## 왜 384?

- 224px에서는 구조물의 미세한 균열, 기울기 등 디테일이 손실될 수 있음
- 384px로 키우면 더 세밀한 패턴 인식 가능
- ConvNeXt는 다양한 해상도에서 잘 동작하는 것으로 알려져 있음

## 결과

| Fold | Val LogLoss | Val Acc |
|------|------------|---------|
| 1 | 0.0615 | 99.6% |
| 2 | 0.0625 | 99.1% |
| 3 | 0.0652 | 99.6% |
| 4 | 0.0639 | 99.6% |
| 5 | 0.0579 | 100% |
| **Mean** | **0.0622** | |

- 추론: 5-Fold x 5-TTA = 25개 예측 평균
- 추가: exp007(224) + exp008(384) 다중 해상도 앙상블도 생성

## 비교

| 실험 | 해상도 | 배치 | CV LogLoss |
|------|--------|------|------------|
| exp006 | 224 | 8x4=32 | 0.0522 |
| **exp008** | **384** | **4x8=32** | **0.0622** |

## 관찰

- 384 해상도가 224보다 CV가 약간 나쁨 (0.0622 vs 0.0522)
- 데이터 1,100개로는 큰 해상도의 이점을 충분히 활용 못함
- 30 에폭 내에 완전히 수렴하지 못한 fold도 있음 (Fold 5는 마지막까지 개선 중)
- 다만 224와 384 예측이 83개 샘플에서 크게 다름 → 다중 해상도 앙상블에 가치 있음
